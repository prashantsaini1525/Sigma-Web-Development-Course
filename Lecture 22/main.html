Width and Height in px

When you set width and height using pixels (px), you target a specific screen size exactly. If someone views the page on a screen that isn’t exactly that size, the layout can break or overflow.

Hinglish: Jab hum px use karke width/height set karte hain, to hum ek fixed size target kar rahe hote hain. Agar user ki screen us exact size se match nahi karti, to page ka layout toot sakta hai ya scroll bar aa sakta hai.

Width and Height in vw & vh

Using viewport units—vw (viewport width) and vh (viewport height)—lets elements automatically adapt to the user’s screen. For example, 100vh means “take up 100% of the viewport’s height.” Likewise, 100vw means “take up 100% of the viewport’s width.” This ensures that as the browser window changes size, the element scales accordingly.

Hinglish: vw aur vh use karne ka matlab hai page element screen ke hisab se adjust ho jayega. 100vh ka matlab hai poori screen height le lo, aur 100vw ka matlab hai poori screen width. Jaise hi screen size badlegi, element apne aap resize ho jayega.

Font Size with em

The em unit is relative to the font size that the element would normally inherit. If the browser’s default font-size is 16px, setting font-size: 2em; makes the text 32px. It essentially multiplies the inherited size by the given factor. 


1em = 16px
2em = 32px
0.5em = 8px

Hinglish: em unit element ke inherited font size par depend karta hai. Agar default font-size 16px hai aur aap font-size: 2em; likhte ho, to text 32px ho jayega (16px × 2). Isi tarah, 0.5em ka matlab hai aadha size (16px × 0.5 = 8px).

Font Size with rem

The rem unit is always relative to the root (html) element’s font size. If you set html { font-size: 12px; }, then 1rem = 12px, no matter where you use it in the document. This makes it easier to maintain consistent sizing throughout the page. 


1rem = 12px
2rem = 24px
3rem = 36px

Hinglish: rem hamesha root element (html) ke font size par depend karta hai. Agar aapne html { font-size: 12px; } set kiya hai, to 1rem = 12px, chahe kahin bhi use karo. Isse aapko pure page me consistent font sizing milti hai.

Width and Height in vmin

The vmin unit stands for “viewport minimum.” It uses whichever viewport dimension (width or height) is smaller. For instance:

  • Phone in portrait mode: The viewport height is larger than its width, so 1vmin equals 1% of the viewport width. Thus, 100vmin = 100% of viewport width.

    Hinglish: Phone portrait me height zyada hoti hai aur width kam, to vmin width (chhota side) pe base karega. Matlab 100vmin = 100% of width.

  • Phone in landscape mode: The viewport width is larger, so 1vmin equals 1% of the viewport height. Thus, 100vmin = 100% of viewport height.

    Hinglish: Landscape me width zyada aur height kam, to vmin height (chhota side) pe base karega. Matlab 100vmin = 100% of height.

  • Laptop or desktop: Typically, screens are wider than they are tall, so vmin equals a percentage of the viewport height. If your window is perfectly square (width = height), then 100vmin is the full width and full height.

    Hinglish: Laptop/desktop me aksar width zyada hoti hai, to vmin height pe base karega (chhota side). Agar window square hai, to 100vmin dono (width aur height) le lega.

Using vmin is a great way to create elements that maintain a consistent shape (for example, a perfect square) across all devices. If you want a box that’s always half the size of the smaller viewport dimension, you can simply write width: 50vmin; height: 50vmin;, and it will scale appropriately on both phones and laptops.

Hinglish: vmin se aap square element ya koi aisa element bana sakte ho jo chhote ya bade screen pe hamesha same shape aur ratio maintain kare. Agar aap 50vmin set karte ho, to box hamesha chhote viewport dimension (width ya height, jo bhi chhota hai) ka 50% lega. Mobile ho ya laptop, element automatically adjust ho jayega.
